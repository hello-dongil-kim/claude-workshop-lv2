# Block 1: Hooks — 자동으로 걸리는 품질 체크 장치

## EXPLAIN

### Hooks란?

Claude의 작업 흐름 **특정 시점에 자동 실행**되는 쉘 스크립트.

**비유:** 문서 최종 저장 전 자동 맞춤법 검사. 사람이 매번 체크하지 않아도, 시스템이 알아서 확인해준다.

### Hook 실행 시점 4가지

> 📌 **변동 정보 (최종 확인: 2026-06-14)** — Hook 이벤트 이름은 바뀔 수 있습니다. 아래 4가지는 비개발자가 가장 자주 쓰는 시점만 추린 것이고, 이 외에도 약 9가지(UserPromptSubmit, Notification, SubagentStop, SessionStart, SessionEnd 등) 이벤트가 있습니다. 전체 목록과 정확한 이름은 공식 문서(https://docs.claude.com)를 확인하세요.

| Hook | 시점 | 기획자 활용 예시 |
|------|------|-----------------|
| **PreToolUse** | 도구 실행 **직전** | 중요 파일 삭제 차단 |
| **PostToolUse** | 도구 실행 **직후** | 산출물 포맷 자동 검증 |
| **PreCompact** | 컨텍스트 압축 **직전** | 핵심 맥락 보존 지시 |
| **Stop** | Claude가 응답을 마쳤을 때 | 산출물 자동 저장/로깅 |

### Hooks vs CLAUDE.md — 가장 중요한 차이

같은 "규칙"처럼 보이지만 강제력이 완전히 다르다.

| 구분 | CLAUDE.md | Hooks |
|------|-----------|-------|
| 성격 | 조언·가이드 | **명령·규칙** |
| 준수율 | 대체로 따르지만 무시될 수 있음 | **반드시 실행됨** |
| 용도 | 스타일, 선호사항 | 포맷팅, 보안, 알림 |

> **핵심 원칙:** "지켜주면 좋겠다"는 CLAUDE.md에, **"반드시 지켜야 한다"는 Hooks로**.

> **⚠️ 안전 주의:** Hook은 반드시 실행되는 만큼, 잘못 만들면 모든 작업이 막히거나(차단형 Hook이 exit code 2를 내면 도구 실행 자체가 막힙니다) 매번 무겁게 돌 수 있습니다. 처음에는 **좁은 범위로** 시작하고, 의도대로 동작하는지 확인한 뒤 넓혀가세요. 새 Hook은 **먼저 개인용 설정(settings.local.json)에서 로컬로 검증**한 뒤, 팀 공유(settings.json)로 올리는 것이 안전합니다.

### 비기술 직군에게 유용한 Hook 4대 활용 사례

| 상황 | Hook이 하는 일 |
|------|---------------|
| **위험 명령 차단** | `rm -rf` 같은 파일 삭제 명령을 자동으로 막음 |
| **작업 완료 알림** | Claude가 작업을 마치면 데스크톱 알림 표시 |
| **자동 포맷팅** | 문서 저장 시 자동으로 서식 통일 |
| **민감 정보 차단** | 주민번호·계좌번호 패턴이 감지되면 작업 중단 |

> **참고:** Hooks는 IT 관리자의 도움을 받아 설정하는 경우가 많다. 비기술 직군은 **개념과 활용 사례를 이해**하는 것이 우선이고, JSON 작성은 Claude에게 맡기면 된다.

### Hooks를 다른 도구와 구분하기 — MCP vs Hooks

이번 세션은 MCP와 Hooks를 함께 다룬다. 둘 다 "Claude의 행동을 바꾼다"는 점에서 비슷해 보이지만 역할이 다르다.

| 구분 | MCP | Hooks |
|------|-----|-------|
| 역할 | **행동 반경 확장** — 읽고, 쓰고, 연결하기 | **품질 자동화** — 실수 방지, 일관성 보장 |
| 비유 | 사무실에 도구 설치 | 도구 사용 전후에 자동 체크리스트 |
| 기획자 관점 | "어떤 도구에 접근할 수 있는가" | "내가 매번 확인하던 것을 시스템으로 거는 것" |

### 실전 예시

**가장 간단한 Hook — 파일 생성 시 알림:**

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "hooks": [
          {
            "type": "command",
            "command": "echo '✓ 파일이 생성되었습니다'"
          }
        ]
      }
    ]
  }
}
```

- `"matcher": "Write"` → Write 도구가 실행된 직후에만 작동
- 터미널에 확인 메시지 출력

> **핵심:** 이 JSON 문법을 외울 필요 없다. Claude에게 **"파일 생성할 때마다 알림 뜨는 Hook 설정해줘"**라고 말하면 settings.json에 자동으로 추가해준다. 결과만 검토하면 된다.

## QUIZ

**문제 1 (회상형):** Hooks의 핵심 역할을 가장 잘 설명한 것은?

| 선택지 | 내용 |
|--------|------|
| A | 외부 서비스에 데이터를 전송하는 통로 |
| B | Claude가 작업하기 전후에 자동으로 실행되는 품질 체크 |
| C | settings.json의 설정을 자동으로 동기화하는 시스템 |

**정답:** B — Hooks는 "자동 체크리스트". 사람이 매번 확인하지 않아도, 특정 시점에 시스템이 알아서 검증·저장·차단해준다.
- A 오답: 외부 서비스 연결은 MCP의 역할이다 — Hook은 작업 흐름의 특정 시점에 거는 자동 동작이다.
- C 오답: Hook은 설정을 동기화하지 않는다 — 정해진 시점에 쉘 명령을 실행할 뿐이다.

**문제 2 (적용형):** 다음 상황에서 가장 적절한 것은? — "보고서는 항상 한국어로" 같은 건 대체로 잘 지켜지면 충분하지만, "주민번호·계좌번호가 들어간 파일은 반드시 작업이 중단돼야 한다"는 한 번도 새면 안 된다.

| 선택지 | 내용 |
|--------|------|
| A | 둘 다 CLAUDE.md에 적는다 |
| B | "한국어"는 CLAUDE.md에, "민감정보 차단"은 Hook(PreToolUse)으로 건다 |
| C | 둘 다 Hook으로 건다 |

**정답:** B — "지켜주면 좋겠다"는 CLAUDE.md(조언), "반드시 지켜야 한다"는 Hook(강제)이다. 민감정보 차단처럼 절대 새면 안 되는 규칙은 반드시 실행되는 Hook이 맞다.
- A 오답: CLAUDE.md는 무시될 수 있어 "절대 차단"에는 부적합하다.
- C 오답: 단순 스타일 규칙까지 Hook으로 걸면 과하고, 잘못 만들면 매 작업이 무겁거나 막힐 수 있다.
