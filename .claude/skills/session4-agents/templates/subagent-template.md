# Subagent 템플릿

아래 템플릿을 복사하여 `.claude/agents/{에이전트명}/SKILL.md` 파일로 저장하세요.
`{...}` 부분을 자신의 업무에 맞게 채우면 됩니다.

---

## 템플릿

```markdown
---
name: {에이전트-이름}
description: |
  {에이전트 설명. 어떤 상황에서 활성화되는지 구체적으로.}
  예: "데스크 리서치 전문 에이전트. '리서치해줘', '조사해줘', '시장 분석' 키워드에 트리거."
model: sonnet
maxTurns: 15
tools:
  - WebSearch
  - Read
  - Write
---

## 역할
당신은 {역할}입니다.
{이 에이전트가 전문으로 하는 일을 1~2문장으로 기술}

## 작업 원칙
- {원칙 1: 예) 사실(Fact)과 의견(Opinion)을 명확히 구분}
- {원칙 2: 예) 출처를 반드시 표기 (URL 또는 문서명)}
- {원칙 3: 예) 핵심 발견 3~5개를 먼저 제시하고, 상세 내용은 뒤에 배치}

## 산출물 형식
{산출물 저장 경로와 포맷 설명}

예시:
### [주제명] 리서치 요약
- **핵심 발견**: 3~5개 불릿
- **상세 분석**: 발견별 근거와 데이터
- **출처 목록**: 참고한 자료 리스트
- **한계**: 이번 조사에서 다루지 못한 부분
```

---

## 작성 가이드

### 1. name (에이전트 이름)
- `@에이전트명`으로 호출할 이름
- 영문 소문자, 하이픈(`-`)만 사용
- 역할이 직관적으로 드러나도록: `desk-researcher`, `meeting-scribe`, `ux-reviewer`

### 2. description (트리거 조건)
- **가장 중요한 필드** — 이 내용을 기반으로 Claude가 자동 트리거 여부를 판단
- 에이전트가 "무엇을 하는지" + "어떤 키워드에 반응하는지" 모두 포함
- 키워드는 사용자가 실제 대화에서 쓸 법한 단어로 선정

### 3. model (사용 모델)
| 모델 | 용도 | 비용 |
|------|------|------|
| `opus` | 복잡한 판단, 전략 수립, 고품질 문서 | 높음 |
| `sonnet` | 리서치, 요약, 데이터 정리 (추천 기본값) | 중간 |
| `haiku` | 단순 반복, 포맷 변환, 분류 | 낮음 |

**원칙:** Subagent는 보조 역할이므로, 대부분 `sonnet`이면 충분하다.

### 4. maxTurns (최대 턴 수)
- 에이전트가 자율적으로 수행할 수 있는 최대 턴 수
- 간단한 작업: `5~10` / 리서치 등 탐색 작업: `10~20` / 복잡한 분석: `20~30`

### 5. tools (도구 목록)
| 도구 | 용도 |
|------|------|
| `WebSearch` | 웹 검색 (리서치 에이전트에 필수) |
| `Read` | 파일 읽기 (기존 문서 참조 시) |
| `Write` | 파일 쓰기 (산출물 저장 시) |
| `Edit` | 파일 수정 (기존 문서 업데이트 시) |
| `Bash` | 터미널 명령 실행 (데이터 처리 시) |

필요한 도구만 최소한으로 부여하는 것이 안전하다.

## 기획자용 에이전트 아이디어

| 에이전트명 | 역할 | 주요 도구 |
|------------|------|-----------|
| `desk-researcher` | 시장/경쟁사/트렌드 리서치 | WebSearch, Read, Write |
| `meeting-scribe` | 회의록 정리 및 액션 아이템 추출 | Read, Write |
| `data-formatter` | 데이터 정리, 포맷 변환, 테이블 구조화 | Read, Write |
| `ux-reviewer` | 사용성 체크리스트 기반 리뷰 | Read, Write |
| `spec-drafter` | PRD/스펙 문서 초안 작성 | Read, Write |
| `competitor-scout` | 경쟁사 동향 모니터링 및 요약 | WebSearch, Read, Write |
