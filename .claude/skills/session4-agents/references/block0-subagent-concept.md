# Block 0: Subagent 개념

## EXPLAIN

### Subagent란?

Claude Code에서 대화를 나누다 보면, 중간에 리서치나 파일 정리 같은 **보조 작업**이 필요할 때가 있다. 이 보조 작업을 메인 대화에서 직접 하면 어떻게 될까?

- 중간 과정(검색 결과, 임시 메모, 시행착오)이 메인 대화에 쌓인다
- 컨텍스트 윈도우가 빠르게 소진된다
- 핵심 맥락이 밀려나면서 Claude의 응답 품질이 떨어진다

**Subagent**는 이 문제를 해결한다:
- **별도의 컨텍스트 윈도우**에서 독립적으로 작업
- 작업이 끝나면 **결과만** 메인 대화로 돌려줌
- 메인 대화의 맥락은 깨끗하게 유지

### 비유: 기획 리더와 리서치 보조원

| 역할 | 하는 일 | 대화 상대 |
|------|---------|-----------|
| **기획 리더** (메인 Claude) | 전체 방향 설정, 의사결정, 산출물 작성 | 사용자 (나) |
| **리서치 보조원** (Subagent) | 특정 주제 조사 → 요약본 작성 → 리더에게 전달 | 기획 리더 |

기획 리더가 "이 주제 조사해서 요약본만 줘"라고 지시하면, 보조원은 자기 자리에서 조사를 마치고 **A4 한 장 요약본**만 전달한다. 리더의 책상 위에는 중간 과정의 스크랩 자료가 쌓이지 않는다.

### 핵심 가치: 컨텍스트 오염 방지

Subagent의 가장 큰 가치는 **메인 대화의 컨텍스트를 오염시키지 않는 것**이다.

| 방식 | 컨텍스트 소비 | 메인 대화 영향 |
|------|---------------|----------------|
| 메인에서 직접 작업 | 높음 (중간 과정 전부 포함) | 맥락 희석, 품질 저하 |
| Subagent 위임 | 낮음 (결과 요약만 반환) | 맥락 보존, 품질 유지 |

### Subagent 호출 방식 3가지

| 방식 | 사용법 | 설명 |
|------|--------|------|
| **@에이전트명** | `@desk-researcher 이 주제 조사해줘` | 대화 중 특정 에이전트를 이름으로 호출 |
| **/agents** | 터미널에서 `/agents` 입력 | 에이전트를 만들고 편집·삭제·관리하는 UI. 특히 "Generate with Claude"로 비개발자도 손쉽게 새 에이전트를 만들 수 있다 |
| **자동 위임** | 메인 대화에서 자동 호출 | 내가 설명한 작업과 각 Subagent의 `description`이 매칭되면, Claude가 메인 대화 중 상시 자동으로 해당 Subagent에 작업을 위임한다 |

### 내장 Subagent

Claude Code에는 기본으로 내장된 Subagent가 있다:

| 에이전트 | 모델 | 역할 | 기획자 활용 예시 |
|----------|------|------|------------------|
| **Explore** | Haiku (빠름) | 파일 구조 탐색, 코드베이스 검색 (읽기 전용) | "이 프로젝트 폴더 구조를 파악해줘" |
| **Plan** | 상속 | 계획 수립 시 코드베이스 연구 (읽기 전용) | Plan mode에서 자동 사용 |
| **general-purpose** | 상속 | 탐색 + 수정이 모두 필요한 복합 작업 | 복잡한 다단계 작업 위임 |
| **claude-code-guide** | Haiku | Claude Code 공식 문서 조회 | "CLAUDE.md 문법 규칙이 뭔지 알려줘" |

이 내장 에이전트들은 별도 설정 없이 바로 사용할 수 있다. Claude가 작업에 따라 자동으로 적절한 에이전트를 선택한다.

### 한 단계 더 — Cowork 탭으로 백그라운드 자율 실행

여기까지가 **메인 대화 안에서** 작동하는 Subagent다. 데스크톱 앱에는 한 단계 더 나아간 형태가 있다.

**Cowork 탭**은 작업을 지시하면 Claude가 **클라우드 VM에서 독립적으로 실행**한다:

- 작업을 지시하고 탭을 닫아도 백그라운드에서 계속 실행
- 완료되면 알림으로 결과 확인
- 긴 시간이 걸리는 리서치, 대량 파일 처리에 적합

| 비교 | Subagent (Code 탭) | Cowork 탭 |
| ---- | ------------------ | --------- |
| 실행 위치 | 내 컴퓨터 (별도 컨텍스트) | 클라우드 VM (독립 환경) |
| 메인 대화 점유 | 결과만 반환 | 탭 자체가 분리됨 |
| 실행 시간 | 짧은~중간 작업 | 긴 작업도 OK (내가 자리를 비워도 진행) |
| 활용 예 | 파일 탐색, 리서치 요약 | 대용량 데이터 분석, 다단계 자동화 |

> **핵심 차이:** Subagent는 "메인 대화 안에서 결과만 받기", Cowork는 "탭을 닫아두고 나중에 결과 확인". 작업 길이와 자리 점유 여부로 선택한다.

## EXECUTE

다음을 직접 실행해보세요:

**Step 1: 내장 에이전트 목록 확인**
```
/agents
```
- 현재 사용 가능한 에이전트 목록을 확인한다.

**Step 2: Explore 에이전트 체험**
```
이 폴더의 파일 구조를 탐색해줘. Explore 에이전트를 사용해.
```
- Explore 에이전트가 별도로 작업한 뒤 결과만 돌려주는 것을 확인한다.
- 메인 대화에 중간 탐색 과정이 쌓이지 않는 점에 주목한다.

## QUIZ

**문제:** Subagent를 사용하는 가장 큰 이유는?

| 선택지 | 내용 |
|--------|------|
| A | 작업 속도가 빨라져서 |
| B | 메인 대화의 컨텍스트를 오염시키지 않기 위해 |
| C | 여러 모델을 동시에 사용할 수 있어서 |

**정답:** B — Subagent는 별도 컨텍스트 윈도우에서 작업하고 결과만 반환하므로, 메인 대화의 맥락이 중간 과정으로 오염되지 않는다. 이것이 Subagent의 핵심 가치다.
