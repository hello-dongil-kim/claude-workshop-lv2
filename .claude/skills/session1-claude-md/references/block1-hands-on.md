# Block 1: CLAUDE.md 실습

## EXPLAIN

### /init으로 시작하기

처음부터 빈 파일에 작성하기 어려울 때, `/init` 명령어가 도움이 된다:
- 프로젝트 구조를 분석해서 CLAUDE.md 초안을 자동 생성
- 이걸 기반으로 불필요한 줄을 삭제하고, 나에게 맞게 다듬으면 됨

### 핵심 워크플로우 — 4단계 작업법

> **🔁 Lv.1 복습 + 확장:** 4단계 워크플로우는 Lv.1에서 한 번 체험한 패턴이다. Lv.2에서는 CLAUDE.md가 적용된 환경에서 같은 패턴을 다시 해보면서, **규칙이 자동 적용되는 효과**를 체감하는 것이 목적이다.

CLAUDE.md를 작성한 뒤, 실제 작업은 이 4단계 패턴을 따르면 효과적이다:

| 단계 | 이름 | 하는 일 | 권한 모드 |
|------|------|---------|----------|
| **1단계** | 탐색 (Explore) | 현재 상태 파악 | Plan mode |
| **2단계** | 계획 (Plan) | 작업 전략 수립, 검토·수정 | Plan mode |
| **3단계** | 실행 (Implement) | 계획대로 파일 생성·수정 | Ask / Auto accept |
| **4단계** | 정리 (Commit) | 결과 확인 및 저장 | Ask / Auto accept |

**1단계 — 탐색:** Plan mode에서 현재 상태를 파악한다. 읽기만 하므로 안전하다.

```
"input/ 폴더에 어떤 파일들이 있는지 분석하고,
각 파일의 구조와 내용을 요약해줘"
```

**2단계 — 계획:** Claude에게 작업 계획을 먼저 세우게 하고 검토한다.

```
"이 데이터를 기반으로 경쟁사 비교 보고서를 만들 건데,
작업 계획을 먼저 세워줘"
```

**3단계 — 실행:** 모드를 전환하고, 기대하는 결과 형태를 미리 알려준다.

```
"output/경쟁사분석.md를 만들어줘.
각 경쟁사별로 '제품명 | 가격 | 주요기능 | 차별점' 표와,
마지막에 5줄 이내 종합 요약"
```

> **diff 뷰어로 변경 확인:** 파일이 수정되면 IDE에서 변경 전/후를 시각적으로 비교하는 diff 뷰어가 자동으로 표시된다. 무엇이 바뀌었는지 한눈에 확인하고 승인/거부할 수 있다.

**4단계 — 정리:** 결과를 확인하고 보완점을 파악한다.

```
"생성된 파일 목록 정리해주고, 추가로 보완하면 좋을 부분 알려줘"
```

> **핵심:** 복잡한 작업일수록 Plan mode로 먼저 탐색하고, 계획을 세운 뒤 실행하는 것이 안전하고 효율적이다.

### 비개발자용 CLAUDE.md 템플릿

아래는 비개발 직군에 최적화된 CLAUDE.md 구조다:

```markdown
# Project: [프로젝트명]
[한 줄 프로젝트 설명]

## 나의 역할과 맥락
- 직무: UX 리서처 / 서비스 기획자
- 주요 이해관계자: [팀, 유관부서]
- 현재 프로젝트 단계: Discovery / Define / Develop / Deliver

## 산출물 규칙
- 문서 언어: 한국어 (업무용 존댓말)
- 기본 포맷: Markdown
- 파일 저장 위치: ./outputs/
- 네이밍: YYYY-MM-DD [산출물유형]_v[버전]

## 작업 원칙
- 이론보다 실무 예시 우선
- 제안 시 항상 선택지 + 추천안 + 리스크 분석 포함
- 리서치 결과는 반드시 출처와 신뢰도 표기

## 절대 하지 말 것
- 임의로 파일 삭제하지 않기
- 확인 없이 외부 API 호출하지 않기
```

### 템플릿의 설계 의도

| 섹션 | 왜 필요한가 |
|------|-------------|
| 역할과 맥락 | Claude가 "누구에게" 답하는지 파악 → 톤과 깊이 조절 |
| 산출물 규칙 | 매번 "한국어로 써줘" 반복 방지 |
| 작업 원칙 | 기획자가 원하는 응답 스타일 고정 |
| 절대 하지 말 것 | 치명적 실수 방지 (파일 삭제, 무단 API 호출 등) |

## EXECUTE

다음을 직접 실행해보세요:

**Step 1: /init으로 자동 생성**
```
/init
```
- 생성된 CLAUDE.md를 읽어보고, 불필요한 줄을 체크

**Step 2: Claude에게 커스텀 요청**

```text
이 CLAUDE.md를 내 상황에 맞게 수정해줘.
- 직무: [자신의 직무 입력]
- 프로젝트: [프로젝트명 입력]
- 산출물 규칙: 한국어, 마크다운, ./outputs/ 폴더에 저장
@templates/claude-md-template.md 를 참고해서.
```

- Claude가 수정한 결과를 검토하고, 마음에 안 드는 부분만 추가 지시
- **핵심: 직접 다 쓸 필요 없다. Claude에게 초안을 시키고, 내가 검토·수정하면 된다**

**Step 3: 테스트**
- 새 세션을 열고 (IDE: 패널에서 New Chat / 터미널: 새 탭 → `claude`)
- `"내가 누군지, 어떤 규칙을 따르는지 알려줘"` 입력
- CLAUDE.md의 내용을 잘 반영하는지 확인

**Step 4: 4단계 워크플로우 체험**
- Plan mode로 전환 (IDE: 모드 드롭다운 / 터미널: Shift+Tab)
- `"이 프로젝트 폴더 구조를 분석해줘"` → 탐색 단계 체험
- `"이 구조를 개선할 계획을 세워줘"` → 계획 단계 체험
- Ask permissions로 전환하고 간단한 파일 생성 실행 → 실행 단계 체험

## QUIZ

**문제:** CLAUDE.md의 구조로 가장 적합한 순서는?

| 선택지 | 순서 |
|--------|------|
| A | HOW → WHAT → WHY |
| B | WHY → HOW → WHAT |
| C | WHAT → WHY → HOW |

**정답:** C — WHAT(프로젝트 개요) → WHY(목적과 맥락) → HOW(작업 방식과 규칙). Claude가 큰 그림부터 세부 규칙 순서로 이해해야 일관된 응답을 만든다.
