# Block 2: CLAUDE.md 심화

## EXPLAIN

### 실전 팁 3가지

**1. 하위 디렉토리 CLAUDE.md**
- 프로젝트 루트뿐 아니라 하위 폴더에도 CLAUDE.md를 배치할 수 있음
- 예: `research/CLAUDE.md`는 리서치 폴더에서 작업할 때만 추가로 읽힘
- 여러 프로젝트가 하나의 폴더에 함께 있는 대규모 프로젝트에서 유용

```
project/
├── CLAUDE.md              ← 공통 규칙
├── research/
│   └── CLAUDE.md          ← 리서치 작업 시 추가 규칙
└── planning/
    └── CLAUDE.md          ← 기획 작업 시 추가 규칙
```

**2. 조건부 규칙**
```markdown
<important if="파일을 생성하거나 수정할 때">
반드시 ./outputs/ 디렉토리에 저장할 것.
파일명은 YYYY-MM-DD [산출물유형]_v[버전].md 형식을 따를 것.
</important>
```
- 특정 조건에서만 강조되는 규칙을 표시
- 항상 적용되는 규칙과 상황별 규칙을 구분

**3. 팀 공유 전략**

| 방법 | 설명 |
|------|------|
| Git 커밋 | CLAUDE.md를 레포에 포함 → 팀 전원이 동일 규칙 사용 |
| 개인 규칙 분리 | `~/.claude/CLAUDE.md`는 개인 전용 (모든 프로젝트 공통) |
| 프로젝트 + 개인 조합 | 프로젝트 CLAUDE.md (팀 공통) + 개인 CLAUDE.md (개인 스타일) |

### 앞으로 배울 것: 전체 확장 기능 계층 구조

CLAUDE.md는 Claude Code 확장의 **기초 레이어**다. 이후 세션에서 위로 한 단계씩 쌓아 올린다:

```
Plugins (배포·패키징)               ← Workshop
  └ Agent Teams (병렬 협업)          ← Session 4
    └ Subagents (독립 위임)          ← Session 4
      └ Hooks (자동화) + MCP (도구 연결) ← Session 3
        └ Skills (자동 로딩 지식)      ← Session 2
          └ CLAUDE.md (기본)          ← 지금 여기
```

지금 가장 아래층을 완성했다. 다음 세션부터 하나씩 올라간다.

### 효과적인 프롬프트 패턴 3종

CLAUDE.md가 "규칙"이라면, 프롬프트는 "지시"다. 좋은 프롬프트 패턴을 익히면 결과 품질이 크게 달라진다.

**패턴 1: 역할 + 작업 + 형식**

```
"시장 분석가 역할로, input/data.csv를 분석해서
경쟁사 비교표(표 형식)를 output/에 생성해줘"
```

**패턴 2: 제약 조건 명시**

```
"보고서를 작성하되:
- 5페이지 이내
- 각 섹션 10줄 이하
- 수치에 출처 표기
- 한국어로"
```

**패턴 3: 검증 포함**

```
"보고서를 만들고, 다 만든 후에 확인해줘:
- 모든 수치에 출처가 있는지
- 표 형식이 통일되어 있는지
- 오탈자가 없는지"
```

> **핵심:** "잘 정리해줘"는 결과가 매번 다르다. 형식·분량·저장 위치를 구체적으로 지정해야 일관된 결과를 얻는다.

### 자동 메모리 시스템

Claude Code는 대화에서 배운 내용을 **자동으로 기억**한다 (사용자 선호, 중요한 발견 등). 매 세션 시작 시 자동 로드된다.

**명시적으로 기억시키기:**

```
"앞으로 보고서 작성할 때 항상 요약을 3줄로 시작해. 이것 기억해줘."
```

**메모리 관리:** `/memory` 명령으로 저장된 항목을 확인·편집할 수 있다. 오래되거나 틀린 내용은 Claude에게 "XX에 대한 기억 삭제해줘"로 제거.

### 비용 관리 팁

| 방법 | 효과 |
|------|------|
| 긴 대화는 `/compact` 또는 `/clear` | 대화 길이에 따라 비용이 빠르게 증가 |
| 구체적인 프롬프트 | "다 찾아봐"보다 "A, B, C를 비교해줘" |
| 서브에이전트에 저렴한 모델 배정 | 단순 작업은 `model: haiku` (Session 4에서 다룸) |
| `/cost`로 수시 모니터링 | 현재 세션 비용을 실시간 확인 |

### 좋은 CLAUDE.md vs 나쁜 CLAUDE.md

| 나쁜 예 | 좋은 예 | 이유 |
|---------|---------|------|
| "잘 해줘" | "선택지 3개 + 추천안 + 리스크 포함" | 구체적 기준 |
| 500줄 상세 매뉴얼 | 핵심 규칙 150줄 이내 | 준수율 |
| "Markdown으로 써줘" | "한국어, 존댓말, ./outputs/에 저장" | 복합 규칙을 한 섹션에 |

## QUIZ

**문제:** 팀에서 CLAUDE.md를 공유할 때, "개인 스타일 규칙"은 어디에 넣어야 하나?

| 선택지 | 위치 |
|--------|------|
| A | 프로젝트 루트의 CLAUDE.md |
| B | ~/.claude/CLAUDE.md |
| C | .claude/settings.json |

**정답:** B — `~/.claude/CLAUDE.md`는 모든 프로젝트에 공통으로 적용되는 개인 전용 규칙. 팀 규칙은 프로젝트 루트 CLAUDE.md에, 개인 스타일은 홈 디렉토리에 분리하는 것이 깔끔하다.
